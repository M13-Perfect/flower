# 店小秘 → Flower 自动取单 · 操作教程

面向使用者的一步步教程。技术细节看 `AGENTS.md`，这里只讲**怎么用**。

---

## 0. 它是干什么的（一句话）

在店小秘订单页点一个「→Flower」按钮 → 订单自动送进本地小程序 → Flower 桌面自动载入并解析 →
你复核后点「生成」出 SVG/DXF/PNG。中间有两道校验，生不出来的会被记进表格，不会把坏单塞进生产。

```
店小秘网页  →[点 →Flower]→  本地服务(收单/去重/写文件)  →  Flower 桌面(自动载入+解析)  →  你点「生成」出件
```

三个部件，都在 `C:\Users\Administrator\Documents\flower\automation\` 下：
- **extension/**：Chrome 扩展（在店小秘页注入按钮）。
- **inbox-service/**：本地小服务（收单、去重、写收件夹、状态台账）。**用前必须先启动它。**
- Flower 桌面端本体在仓库根（`birth_flower_mvp.py`）。

## 1. 首次准备（只做一次）

### 1.1 本地服务（依赖已装好）
依赖已经装在 `inbox-service\.venv`。**换了电脑或删了 .venv** 才需要重装：
```powershell
cd C:\Users\Administrator\Documents\flower\automation\inbox-service
C:\Users\Administrator\Documents\flower\.venv-win\Scripts\python.exe -m venv .venv
.\.venv\Scripts\python.exe -m pip install fastapi uvicorn pydantic sqlalchemy alembic openpyxl httpx pytest
```

### 1.2 扩展：构建 + 加载进 Chrome
```powershell
cd C:\Users\Administrator\Documents\flower\automation\extension
npm install          # 首次或换机才需要
npm run build        # 生成 dist\（每次改了扩展代码都要重跑）
```
然后在 Chrome：
1. 地址栏进 `chrome://extensions`。
2. 右上角打开「**开发者模式**」。
3. 点「**加载已解压的扩展程序**」→ 选文件夹 `C:\Users\Administrator\Documents\flower\automation\extension\dist`。
4. 扩展出现「Flower 取单助手」即成功。（改了代码重新 `npm run build` 后，回这页点扩展的「刷新」图标。）

### 1.3 让 Flower 监听收件夹
用记事本/VSCode 打开仓库根的 `C:\Users\Administrator\Documents\flower\birth_flower_config.json`，
在最外层大括号里加一行（注意逗号别漏）：
```json
  "inbox_folder": "C:\\Users\\Administrator\\Documents\\flower\\outputs\\inbox",
  "inbox_autoparse": true
```
- 路径要和本地服务一致（服务默认就写到这个 `outputs\inbox`，可用下面的 `/healthz` 核对）。
- `inbox_autoparse: true` = 来单自动解析；想只载入不自动解析就改 `false`。
- 不加这两行 = 监听功能关闭，Flower 照常用，不受影响。
- **改完配置要把 Flower 完全关掉重开**才生效。

## 2. 日常使用

### 2.1 启动本地服务（每次开工先做）
**最简单**：双击 `C:\Users\Administrator\Documents\flower\automation\启动服务.bat`。
弹出的黑窗口**保持开着**（关了服务就停）。

手动方式（等价）：
```powershell
cd C:\Users\Administrator\Documents\flower\automation\inbox-service
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8770
```
验证：浏览器开 `http://127.0.0.1:8770/healthz`，看到 `"status":"ok"` 就对了；或点扩展图标看 popup 健康灯变**绿**。

### 2.2 启动 Flower
```powershell
cd C:\Users\Administrator\Documents\flower
.\.venv-win\Scripts\python.exe birth_flower_mvp.py
```

### 2.3 在店小秘发单（单个订单）
1. 打开店小秘，进到订单列表（你平时筛选「待处理」那种页）。
2. 每个订单行右侧会多一个「**→Flower**」按钮；**AI未识别的单按钮是酒红色**。
3. 点某一单的按钮 → 右下角弹「已发送：订单号」即成功。
4. 扩展图标点开有个 **popup**：能看本地服务健康灯（绿=在线）。

> 现在是**列表页每行一个按钮，点哪单发哪单**。

### 2.4 在 Flower 里复核 + 生成
- 发单后，Flower 会**自动把该单的买家定制信息填进备注框并解析**，状态栏显示「📥 新订单已载入：订单号」。
- 你核对识别出来的 月份/花/字体/名字，没问题就点「**生成**」→ 文件出在 `outputs\` 里。
- **扩展和 Flower 都不会自动生成**，生成永远是你手动点。

### 2.5 批量路径（攒一批一起生产，可选）
适合积累了一批订单一次性出件，并自动得到「哪些没生成出来」的表格：
1. 用扩展把多个订单都点「→Flower」（都进了服务的池子）。
2. 让服务导出一个店小秘格式的批量表：
   ```powershell
   curl -X POST http://127.0.0.1:8770/inbox/batch/export
   ```
   返回的 `path` 就是生成的 `outputs\inbox-batches\pooled-*.xlsx`。
3. 在 Flower 用现有的「**导入**」功能选这个 xlsx → 它会批量生成，并产出报告
   `outputs\reports\{批次号}-report.xlsx`（列：订单号/状态/是否需人工核验/原因汇总）。
4. 把状态回写给服务（让台账知道哪些已完成、哪些无法自动生成）：
   ```powershell
   curl -X POST http://127.0.0.1:8770/inbox/batch/sync
   ```
   （服务后台其实也会自动扫这个报告回写，这步是手动催一下。）
5. 报告里「是否需人工核验=是」的，就是**无法自动生成**的单，按 `{批次号}-review.csv` 补字段后重跑。

## 3. 怎么看每单到哪一步了
- 浏览器开 `http://127.0.0.1:8770/inbox/orders` → 列出所有订单和状态。
- 单看一单：`http://127.0.0.1:8770/inbox/orders/订单号`。
- 状态含义：`WRITTEN_TO_INBOX`=已写入收件夹、`QUEUED_FOR_BATCH`=已排进批量、
  `DONE`=已完成、`CANNOT_AUTOGEN`=无法自动生成（带 `reason` 原因）。

## 4. 出问题怎么查

| 现象 | 可能原因 / 处理 |
|---|---|
| 店小秘页面没有「→Flower」按钮 | ① 重新构建后必须回 `chrome://extensions` 点该扩展的**刷新**图标，再刷新店小秘页。② 进店小秘订单列表页后，右下角应弹「Flower 取单助手已就绪 · 本页 N 单」。③ 按 F12 → Console 搜 `[Flower`：能看到「已加载…本页识别订单 N 个」说明脚本在跑——若 N=0 说明本页结构对不上（把该页另存 HTML 发我）；若**完全没有这行日志**，说明没匹配到域名（把地址栏 URL 发我）。 |
| 点按钮提示「无法连接本地服务」 | 本地服务没启动 → 回 2.1 启动；确认 `http://127.0.0.1:8770/healthz` 能开。 |
| 提示「未能识别订单号或定制信息」 | 该订单行结构和已校准的不一样（店小秘改版）→ 把那一单页面另存 HTML 发我，我调 `extension\src\selectors.ts`。 |
| Flower 没自动载入 | `birth_flower_config.json` 的 `inbox_folder` 没配或路径和服务不一致；改完没重开 Flower；确认 `outputs\inbox\订单号.json` 有没有生成。 |
| 端口 8770 被占用 | 启动服务时加 `--port 8799`，并把扩展里 `extension\src\worker\client.ts` 的端口也改一致后 `npm run build`。 |
| 解析出来的 月/花/字体/名字 不对 | 这属于 Flower 的解析质量（尤其「AI未识别」单），在 Flower 里手动改字段即可；与扩展无关。 |
