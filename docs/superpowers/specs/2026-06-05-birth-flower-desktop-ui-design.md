# Birth Flower MVP 桌面应用 UI 设计方案

## 结论

采用“桌面生产工作台”方案。

界面目标不是展示礼品品牌感，而是降低订单处理成本：快速导入备注、解析、人工修正、预览调版、选择输出格式，并在二次确认后生成生产文件。当前项目使用 Tkinter，已有 SVG/DXF/PNG 渲染链路，不更换 UI 框架。

## 项目依据

- `README.md` 说明当前应用是本地桌面 MVP，根据订单备注解析姓名、月份、字体和花朵，人工确认后生成 Birth Flower SVG。
- `ui_app.py` 已有 Tkinter 主窗口、菜单、设置弹窗、备注导入、解析、实时画板、输出格式、人工确认生成。
- `renderer.py` 已有 `render_svg`、`render_dxf`、`render_png`，输出面向雕刻和生产文件。
- `models.py` 中 `EngravingLayout` 固定横向画布尺寸为 `1732 x 1280`，参考图也显示高留白、黑色线稿、右下姓名的横向构图。

## ui-ux-pro-max 取舍

ui-ux-pro-max 的首轮设计系统推荐偏向 `AI Personalization Landing`，该模式面向转化页面，不适合本项目，淘汰。

保留并迁移以下建议：

- Product 类型采用 `Productivity Tool`：工作流清晰、功能色明确。
- Style 采用 `Minimal / Swiss`：网格、留白、低装饰、高可读性。
- Interaction 采用 `Micro-interactions`：解析中、生成中、成功、错误给出轻量反馈。
- UX 采用表单错误与恢复策略：错误靠近字段，异步操作必须有状态，失败必须给出下一步。

## 全局布局

主窗口采用三栏加底部生产栏。

### 左栏：订单与解析

用途：处理输入和人工确认字段。

内容：

- 订单备注输入区：多行文本框，支持长备注粘贴。
- 操作按钮：`导入`、`解析`、`清空`。
- 解析状态：显示 `未解析`、`解析中`、`已解析`、`需人工补齐`。
- 字段区：
  - 姓名/雕刻文本
  - 月份
  - 素材名
  - 字体类型
  - confidence
- warnings 靠近对应字段展示，避免在画板区域常驻大块警告文本。

### 中栏：实时画板

用途：确认最终雕刻构图。

内容：

- 画板固定保持 `1732 x 1280` 横向比例。
- 白色画布，浅灰边界，黑色线稿。
- 花朵控制框使用蓝绿色，文字控制框使用琥珀色。
- 顶部工具条：
  - 适配窗口
  - 100%
  - 重置布局
  - 显示/隐藏辅助框
- 拖动花朵框或文字框只更新预览和布局参数，不触发最终输出。

### 右栏：生产参数

用途：集中放置低频但生成前必须确认的参数。

内容：

- 素材选择：当前匹配素材、候选素材、素材路径状态。
- 字体选择：当前字体、字体路径状态。
- 布局参数：
  - 花 x
  - 花 y
  - 花宽
  - 花高
  - 字 x
  - 字 y
  - 字号
- 高级参数默认折叠，避免干扰高频操作。

### 底部生产栏

用途：最终生成安全门。

内容：

- 输出格式：`SVG`、`DXF` 默认选中，`PNG` 可选。
- 输出路径。
- 状态摘要。
- 主按钮：`人工确认并生成`。

按钮启用条件：

- 姓名不为空。
- 至少选择一种输出格式。
- 输出路径有效且目录可写。
- 素材存在，或已明确使用 fallback。
- 字体存在，或已明确使用系统/fallback 字体。

点击后必须弹出二次确认，列出姓名、素材、字体、格式和输出路径。

## 视觉系统

### 风格

采用生产工具风格，不做营销页、礼品展示页或装饰型编辑器。

原则：

- 高对比。
- 低装饰。
- 信息密度适中。
- 画板优先。
- 控制区紧凑但不拥挤。

### 颜色

- 背景：`#F8FAFC`
- 面板：`#FFFFFF`
- 主文本：`#111827`
- 次级文本：`#4B5563`
- 边框：`#D1D5DB`
- 主操作/花朵选区：`#0D9488`
- 文字选区/注意：`#D97706`
- 错误：`#DC2626`
- 成功：`#16A34A`
- 禁用：`#9CA3AF`

不使用渐变、玻璃拟态、花朵背景图、装饰光斑或大面积单一粉色/紫色主题。

### 字体

界面字体使用系统字体：

- Windows：`Segoe UI`
- macOS：`SF Pro`
- Linux：常见 sans-serif fallback

不下载 Google Fonts，不内置商业字体。雕刻字体仍由用户选择的本地字体文件决定。

### 间距

- 基准单位：8px。
- 面板内边距：12px。
- 区块间距：16px。
- 三栏间距：16px。
- 控件圆角：最多 6-8px。

## 状态与错误

### 空状态

未输入备注时，左栏提示输入或导入订单备注。画板显示默认布局边界和空预览，不显示大段说明。

### 解析中

`解析` 按钮禁用，状态显示 `解析中...`。解析完成后恢复按钮。

### 部分识别

字段缺失时，对应字段下方展示短错误。允许用户手动补齐后继续生成。

### 素材缺失

素材字段下方显示缺失原因，并提供进入设置或重新扫描的入口。最终生成前再次校验。

### 字体缺失

字体字段下方显示缺失原因。若使用 fallback，必须在生成确认弹窗中明确说明。

### SVG 含位图

如果 SVG 素材嵌入 PNG/JPG，生成前提示“这不是纯矢量素材”。用户确认后才能继续。

### PNG 复杂文字

选择 PNG 输出时，如果存在复杂文字排版风险，提示 Pillow/RAQM 支持风险。SVG/DXF 不受该提示阻塞。

### 生成中

主按钮禁用，状态显示当前生成格式。生成失败时保留已成功生成的文件列表，并说明失败格式。

### 生成完成

显示已生成路径，并提供 `打开输出目录`。

## 设置窗口

保留 `文件 -> 设置...`，使用 `ttk.Notebook`。

### 素材库

- 素材目录选择。
- 当前扫描结果列表。
- 当前素材数量。
- 重新扫描。
- 素材缺失或 SVG 解析失败提示。

### 字体库

- 字体文件或字体目录选择。
- 当前扫描结果列表。
- 重新扫描。
- 字体缺失提示。

### AI 识别

- 启用 AI 兜底解析。
- 模型名。
- API Key 环境变量名。
- Project 环境变量名。
- Org 环境变量名。
- 临时 API Key，仅本次运行有效，不写入配置。
- 测试连接。

### 输出默认值

- 默认输出目录。
- 默认输出格式。
- 生成后是否打开输出目录。

### 布局模板

- 保存当前布局为默认。
- 恢复项目默认布局。
- 可扩展为多个模板，但本轮不需要实现多模板管理。

## 交互规则

- 解析结果只预填字段，不自动生成最终文件。
- 任何最终文件输出必须经过 `人工确认并生成`。
- 二次确认弹窗必须显示关键字段。
- warnings 不长期占用画板空间。
- 所有按钮必须有明确 disabled 条件。
- 异步操作必须有状态反馈。
- 错误信息必须给出恢复路径。

## 非目标

- 不更换 Tkinter。
- 不做 Web 版。
- 不做营销落地页。
- 不下载字体。
- 不内置商业字体。
- 不保存明文 API Key。
- 不让 AI 或本地解析自动生成最终文件。
- 不把参考图作为 UI 背景。

## 实施优先级

1. 重排 `ui_app.py` 为三栏加底部生产栏。
2. 把 warnings 改为字段就近提示和底部短状态。
3. 增加画板工具条与控制框显示开关。
4. 增强右栏生产参数区。
5. 增强设置窗口分类。
6. 补充 UI 行为测试。
7. 更新 README。

## 测试建议

- `test_ui_app.py`：验证输出按钮启用条件。
- `test_ui_app.py`：验证未选择格式时拒绝生成。
- `test_ui_app.py`：验证字段缺失时状态提示。
- `test_ui_app.py`：验证 warnings 不依赖常驻 Text 控件。
- `test_renderer.py`：保持 SVG/DXF/PNG 输出测试。
- 手工验证：启动 Tkinter，确认窗口布局、菜单、设置弹窗和画板不重叠。

## 最终判断

综合时间、资金、维护风险和收益，最优方案是在现有 Tkinter 应用内重排为生产工作台。该方案不改变核心依赖，不影响现有渲染链路，能直接降低订单处理误操作和人工确认成本。
