# 生产工作台 · 安装与使用

绿色免安装包。解压后是一个 `工作台\` 目录，三端（开花桌面 / 扫码导入 / 后台服务）共用一份运行时。

## 一、目录结构

```
工作台\
├─ Workbench\
│  ├─ app.exe          ← 双击这个（= 总启动器）
│  ├─ _internal\        运行时与依赖（勿删勿动）
│  ├─ config\
│  │  └─ settings.json  扫码导入端配置（首次需改：素材目录 + EzCad2 路径）
│  └─ data\             所有数据（订单库 inbox.db、收件夹、输出）。升级换 Workbench 时保留本目录即不丢数据
├─ extension-dist\      Chrome 抓单扩展（见《扩展安装.md》）
├─ 安装与使用.md
└─ 扩展安装.md
```

## 二、首次配置（每台机器一次）

1. **扫码导入端**：用记事本打开 `Workbench\config\settings.json`，改两处占位：
   - `material_root` → 本店素材目录；
   - `profiles[0].exe_path` → 本机 `EzCad2.exe` 完整路径。
2. **开花桌面端**：首次启动会在 `Workbench\data\` 自动生成配置，无需手改；素材库/字体库在界面里设置。
3. **抓单扩展**：按《扩展安装.md》在 Chrome 加载（多门店暂用「加载已解压」）。

## 三、日常使用

双击 `Workbench\app.exe` 打开**总启动器**：

- 顶部状态：`● inbox 服务 :8770 运行中`（绿色）= 后台已就绪。启动器会自动在后台拉起服务，无黑窗。
- **🌸 开花桌面**：订单备注→排版→生成 SVG/DXF。
- **🔫 扫码导入**：扫订单号→匹配素材→导入 EzCad2。

关闭总启动器会一并停止后台服务。

## 四、容错（缺依赖也不崩）

- **没开 EzCad2 软件**：扫码导入端点「导入」会提示「请先手动打开雕刻软件」，不崩。
- **后台服务未连接**：状态标黄/红并自动重试；两个按钮仍可用（只是自动取单/退款检查暂不可用）。
- **素材目录为空/不存在**：提示找不到素材，不崩。
- **无扫码枪**：直接手输订单号即可。
- **缺 cairo**：桌面端 PNG 由 Pillow 直接合成、不依赖 cairo；仅末尾爱心/图标等少数矢量装饰在缺 cairo 时自动跳过或退化为纯文字，不影响 SVG/DXF 与主体 PNG 生成。
- **配置文件被手改坏**：扫码导入端 `config\settings.json` 即使被改成非法 JSON，也会回落默认配置、窗口照常打开（不崩）。

## 五、升级

替换 `Workbench\` 里**除 `data\` 与 `config\` 外**的内容（即保留这两个子目录）：
`data\` 存订单库/收件夹/输出与开花桌面配置，`config\settings.json` 存扫码导入端的素材目录/EzCad 路径/排版规则。
两者都保留，升级后无需重配。（最稳：先把 `data\` 与 `config\` 拷出备份，再覆盖其余。）

## 六、数据位置说明

默认数据根 = `Workbench\data\`（exe 同级）。若该目录不可写（如装在受限位置），自动回落到
`%APPDATA%\BirthFlower\`。三端通过环境变量共用同一数据根，无需各自配置。
